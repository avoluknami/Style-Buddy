# Техническое задание (ТЗ)

## Проект: StyleBuddy — веб‑приложение для подбора стиля

---

## 1. Общая информация

**Название проекта:** StyleBuddy
**Тип проекта:** Веб‑приложение (SPA)
**Цель проекта:** Создание пользовательского веб‑приложения для подбора одежды и формирования стильных образов на основе предпочтений пользователя.

**Целевая аудитория:**

* Пользователи 16–35 лет, интересующиеся модой и стилем
* Пользователи, которым нужен быстрый подбор образов

---

## 2. Цели и задачи проекта

### 2.1 Цель

Обеспечить удобный и интуитивно понятный сервис, который помогает пользователю подобрать одежду, сохранить образы и получить рекомендации.

### 2.2 Задачи

* Реализовать регистрацию и авторизацию пользователей
* Реализовать каталог одежды
* Реализовать подбор образов
* Реализовать сохранение избранных образов
* Реализовать адаптивный интерфейс

---

## 3. Функциональные требования

### 3.1 Авторизация и регистрация

* Регистрация пользователя по email и паролю
* Авторизация пользователя
* Выход из системы

### 3.2 Работа с каталогом

* Просмотр списка одежды
* Фильтрация по категориям (верх, низ, обувь, аксессуары)
* Просмотр карточки товара

### 3.3 Подбор образа

* Выбор элементов одежды
* Автоматическое формирование образа
* Просмотр итогового образа

### 3.4 Избранное

* Добавление образов в избранное
* Просмотр сохранённых образов
* Удаление образов из избранного

---

## 4. Нефункциональные требования

* Время загрузки страницы: не более 3 секунд
* Адаптивность под мобильные устройства
* Кроссбраузерность (Chrome, Firefox, Safari)
* Простота и понятность интерфейса

---

## 5. Требования к интерфейсу

* Минималистичный дизайн
* Использование светлой цветовой схемы
* Понятная навигация
* Отзывчивый UI (hover, loading состояния)

---

## 8. Требования к данным и ER-диаграмма

### 8.1 Описание сущностей

#### Пользователь (User)

* id (PK)
* email
* password
* created_at

#### Одежда (ClothingItem)

* id (PK)
* name
* category
* image_url

#### Образ (Outfit)

* id (PK)
* user_id (FK)
* created_at

#### Связь OutfitItems

* outfit_id (FK)
* clothing_item_id (FK)

---

### 8.2 ER-диаграмма базы данных

```
User (1) --------< (M) Outfit
                       |
                       | (M)
                       >-------- (M) ClothingItem
```

* Один пользователь может иметь несколько образов
* Один образ состоит из нескольких элементов одежды
* Один элемент одежды может входить в несколько образов
